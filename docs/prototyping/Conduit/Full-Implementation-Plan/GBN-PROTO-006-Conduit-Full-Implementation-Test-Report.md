from local distributed harness runs, deployed AWS stack checks, mobile-network

## 3. Required Run Metadata

Each live validation run should record:

- stack name
- AWS region
- image tag or image digest set
- test window start and end time
- mobile carrier / network path description
- root `chain_id` values used for bootstrap and upload flows
- authority, receiver, and bridge log artifact paths
- observed bootstrap, ACK, failover, and batch-window timings

## 6. Live Measurement Checklist

Before Phase 10 can be signed off as live-validated, record evidence for:

- first-contact bootstrap success and timing
- returning creator refresh timing
- seed bridge activation and follow-on fanout
- UDP punch / tunnel ACK behavior on the signed port
- upload data forwarding and ACK timing
- bridge failover and reuse behavior under churn
- batch-window behavior
- one end-to-end `chain_id` visible across authority, receiver, bridge, and validation artifacts
